## Spring Bean 생명주기

객체 생성 → 의존성 주입 → 초기화 → 사용 → 소멸의 5단계 과정을 거친다.

> 스프링은 의존성 주입(DI)을 완전히 마친 후에야 초기화 콜백(@PostConstruct)을 호출한다.    
> 따라서 빈 내부에서 DB 연결이나 데이터 로딩 같은 작업을 하려면 생성자가 아니라 초기화 단계에서 수행해야 안전하다.


1. 스프링 컨테이너를 올린다.
2. 생성자를 호출해서 객체 껍데기를 만든다.
3. 의존성 주입을 시작 (@Autowired 걸려있는 것)
4. 초기화 콜백 실행 (@PostConstruct 걸려 있는 애들 이 때 실행)
5. 실제 실행
6. 죽일 때는 @PreDestroy 붙어있는 것들 실행

초기화 단계에서 초기화 콜백을 하는 이유는 빈 껍데기를 호출하지 않고 빈 생성이 완료되었을 때 하기 위함
