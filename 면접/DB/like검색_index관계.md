## like 검색은 index를 타지 않는다? (mysql 기준)

- index는 b-tree구조로 이루어져있고 b-tree구조는 모든 데이터가 첫 글자부터 정렬되어있다.
- 첫글자 기준으로 검색했을 때는 인덱스를 사용할 수 있지만 %(와일드카드)를 사용해 중간부터 처리하는 건 인덱스를 포기하고 풀 스캔을 한다.

- 인덱스를 이용하고 싶다면 컬럼의 타입을 처음부터 full text search 형식으로 저장해야한다.
- full text search를 하게 되면 단어마다 저장하기 때문에 검색 속도가 빨라진다.

- 기존 인덱스보다 저장공간이 더 사용되고 쓰기나 업데이트 시 더 느리다
- 한글인 경우 ngram(한 단어씩)으로 할지 단어기준으로 할지 선택해야한다.
