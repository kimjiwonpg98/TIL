## DI (의존성 주입)

```
객체 지향 프로그래밍에서 객체 간의 결합을 느슨하게 만들기 위해 나온 디자인 패턴으로
한 객체가 실행에 필요한 객체를 생성하고 구현하는게 아니라 주입받아서 사용하도록 하는 것입니다.
```

예를 들어 자동차를 만든다고 했을 때 필요한 부품을 밖에서 만들어서 꽂아주는 것입니다.
나중에 A 엔진에서 B엔진으로 변경한다고 했을 때 A를 B로만 변경해주면 쉽게 수정할 수 있습니다.

장점으로는

1. 결합도가 낮아진다.
  - 추상화된 인터페이스를 의존하기 때문에 사용하는 서비스에서는 수정이 필요하지 않는다.
2. 테스트가 용이하다.
  - inject해서 사용하는 객체를 가짜 객체로 만들어서 응답에 따라 테스트를 처리하기가 쉬워진다.
3. 쉽게 교체할 수 있다.

## AOP (관점 지향 프로그래밍)

```
AOP(관점 지향 프로그래밍)는 애플리케이션의 핵심 비즈니스 로직과 공통 부가 기능(횡단 관심사)을 분리하여 코드의 모듈성을 높이고 중복을 제거하는 방식이다.
```

공통적인 부분을 분리하여 사용하는 쪽에서는 생성된 공통 모듈만 사용할 수 있도록 한다.
